Greedy Pooling Algorithm

How It Works

The Greedy Pooling Engine maximizes truck utilization by filling available capacity with nearby orders, sorted by profit-per-km. At each step, it picks the most profitable order that fits the remaining space — a classic greedy knapsack approach.

Algorithm Steps

1
Sort Available Orders Rank all nearby orders by profit per kilometer of detour: value / detour_km.
2
Check Remaining Capacity Calculate remaining = truck_max - current_load. In this case: 2400 - 1920 = 480 kg.
3
Greedily Fill Pick the top-ranked order that fits. If it fits, load it and repeat. Skip if it exceeds capacity.
4
Output Optimized Route Return the updated manifest with new stops, revised ETA, and projected profit boost.

Pseudocode

// Greedy Truck Pooling function optimizeLoad(truck, orders): orders.sort(by: value/detour_km, desc) remaining = truck.capacity - truck.current_load for order in orders: if order.weight <= remaining: truck. add(order) remaining -= order.weight profit += order.value return truck.manifest, profit

Result: The algorithm increased load from 80% → 100% utilization by adding a 480 kg onion pickup near Angren, generating an extra $340 profit with only 8 km detour.
